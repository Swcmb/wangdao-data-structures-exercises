## 第一章   数据结构 绪论

### 1.1 数据结构的基本概念

1. **数据**：数据是信息的载体，符号的集合、所有能输入到计算机中并能被计算机程序处理的符号的集合，数据是计算机程序加工的原料。
2. **数据元素**：数据的基本单位，通常作为一个整体进行考虑和处理。一个数据元素可由若干数据项组成。
3. **数据项**：构成数据元素的不可分割的最小单位。
4. **数据对象**：数据对象是具有相同性值的数据元素的集合，是数据的一个子集。
5. **数据结构**：数据结构是相互之间存在一种或多种特定关系的数据元素的集合。

**举例需要理解几点:**

1. 学校里的好多类型的表：数据
2. 单独的一张成绩单表：数据对象
3. 成绩单中每一行有姓名、课程、班级、成绩：数据元素
4. 成绩单中每一行的每一个表格姓名等都是一个个的数据项

### 1.2 数据结构的三要素

#### 1.2.1. 数据的逻辑结构

**逻辑结构** 是指数据元素之间的逻辑关系，即从逻辑关系上描述数据。

**逻辑结构包括**：

1. **集合结构**：结构中的数据元素之间除“同属一个集合”外，别无其它关系（例如：一群羊）
2. **线性结构**：结构中的数据元素之间只存在一对一的关系（例如：排队买饭）
3. **树形结构**：结构中的数据元素之间存在一对多的关系（例如：家族树）
4. **图状结构或网状结构**：结构中的数据元素之间存在多对多的关系（例如：交通图）

#### 1.2.2. 数据的存储结构（物理结构）

**存储结构** 是指数据结构在计算机中的表示（又称为映像），包括数据元素的表示和关系的表示。

**存储结构包括**：

1. **顺序存储**：把逻辑上相邻的元素存储在物理位置上也相邻的存储单元中，元素之间的关系由存储单元的邻接关系来体现。
2. **链式存储**：不要求逻辑上相邻的元素在物理位置上也相邻，借助指示元素存储地址的指针来表示元素之间的逻辑关系。
3. **索引存储**：在存储元素信息的同时，还建立附加的索引表，索引表中的每项称为索引项，索引项的一般形式是（关键字，地址）。
4. **散列存储**：根据元素的关键字直接计算出该元素的存储地址，又称哈希存储。

#### 1.2.3. 数据的运算

**数据的运算** 是指对数据元素进行的操作，包括：

1. **插入**：在指定位置添加新元素
2. **删除**：删除指定位置的元素
3. **修改**：修改指定位置元素的值
4. **查找**：查找满足条件的元素
5. **排序**：按某种规则对元素进行排序

#### 1.2.4. 数据类型和抽象数据类型

**数据类型**：是一个值的集合和定义在这个值集上的一组操作的总称。

**抽象数据类型（ADT）**：是指一个数学模型以及定义在该模型上的一组操作。

### 1.3 算法的基本概念

**算法**：是解决特定问题的步骤的描述，是指令的有限序列，其中每一条指令表示一个或多个操作。

**算法的特性**：

1. **有穷性**：算法必须在有限步内结束
2. **确定性**：算法的每一步必须有确切的定义
3. **可行性**：算法的每一步都是可行的
4. **输入**：算法至少有零个输入
5. **输出**：算法至少有一个输出

### 1.4 算法的时间复杂度

**时间复杂度**：是指执行算法所需要的计算工作量，通常用大O符号表示。

**时间复杂度的计算方法**：

1. 找出算法中的基本操作（执行次数最多的操作）
2. 计算基本操作的执行次数的数量级
3. 用大O符号表示

**常见的时间复杂度**：

- O(1)：常数时间复杂度
- O(log n)：对数时间复杂度
- O(n)：线性时间复杂度
- O(n log n)：线性对数时间复杂度
- O(n²)：平方时间复杂度
- O(n³)：立方时间复杂度
- O(2ⁿ)：指数时间复杂度
- O(n!)：阶乘时间复杂度

### 1.5 算法的空间复杂度

**空间复杂度**：是指执行算法所需要的内存空间，通常用大O符号表示。

**空间复杂度的计算方法**：

1. 计算算法所需的存储空间
2. 用大O符号表示

**常见的空间复杂度**：

- O(1)：常数空间复杂度
- O(n)：线性空间复杂度
- O(n²)：平方空间复杂度

## 第二章 线性表

### 2.1 线性表的定义和基本操作

#### 2.1.1 线性表的定义

**线性表**：是具有相同数据类型的n（n≥0）个数据元素的有限序列，其中n为表长，当n=0时称为空表。

**线性表的特点**：

1. 元素的个数有限
2. 元素具有逻辑上的顺序性，在序列中各元素的位置是有序的
3. 元素的数据类型相同
4. 元素之间是一对一的关系

#### 2.1.2 线性表的基础操作

1. **InitList(&L)**：初始化表。构造一个空的线性表L。
2. **Length(L)**：求表长。返回线性表L的长度，即L中数据元素的个数。
3. **LocateElem(L, e)**：按值查找操作。在表L中查找具有给定关键字值的元素。
4. **GetElem(L, i)**：按位查找操作。获取表L中第i个位置的元素的值。
5. **ListInsert(&L, i, e)**：插入操作。在表L中第i个位置上插入指定元素e。
6. **ListDelete(&L, i, &e)**：删除操作。删除表L中第i个位置的元素，并用e返回删除元素的值。
7. **PrintList(L)**：输出操作。按顺序输出线性表L的所有元素值。
8. **Empty(L)**：判空操作。若L为空表，则返回true，否则返回false。
9. **DestroyList(&L)**：销毁操作。销毁线性表，并释放线性表L所占用的内存空间。

### 2.2 顺序表

#### 2.2.1 顺序表的概念

**顺序表**：是用一组地址连续的存储单元依次存储线性表中的数据元素，从而使得逻辑上相邻的两个元素在物理位置上也相邻。

**顺序表的特点**：

1. 随机存取：可以通过下标直接访问任意元素
2. 存储密度高：不需要额外的存储空间来存储元素之间的关系
3. 插入和删除操作需要移动元素，时间复杂度为O(n)

#### 2.2.2 顺序表的实现

**顺序表的结构定义**：

```c
#define MaxSize 100  // 定义最大长度
typedef struct {
    ElemType data[MaxSize];  // 数组存储元素
    int length;  // 当前长度
} SqList;  // 顺序表类型定义
```

#### 2.2.3 顺序表的基本操作

1. **初始化顺序表**：

```c
void InitList(SqList &L) {
    L.length = 0;  // 初始化长度为0
}
```

2. **插入元素**：

```c
bool ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length + 1)  // 插入位置不合法
        return false;
    if (L.length >= MaxSize)  // 表满
        return false;
    for (int j = L.length; j >= i; j--)  // 从后往前移动元素
        L.data[j] = L.data[j-1];
    L.data[i-1] = e;  // 插入元素
    L.length++;  // 长度加1
    return true;
}
```

3. **删除元素**：

```c
bool ListDelete(SqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length)  // 删除位置不合法
        return false;
    e = L.data[i-1];  // 保存删除的元素
    for (int j = i; j < L.length; j++)  // 从前往后移动元素
        L.data[j-1] = L.data[j];
    L.length--;  // 长度减1
    return true;
}
```

4. **按值查找**：

```c
int LocateElem(SqList L, ElemType e) {
    for (int i = 0; i < L.length; i++)
        if (L.data[i] == e)
            return i + 1;  // 返回位序
    return 0;  // 查找失败
}
```

### 2.3 线性表的链式表示

#### 2.3.1 单链表的基本概念

**单链表**：是通过一组任意的存储单元来存储线性表中的数据元素，每个节点包含数据域和指针域。

**单链表的特点**：

1. 非随机存取：只能通过指针依次访问元素
2. 插入和删除操作不需要移动元素，时间复杂度为O(1)
3. 需要额外的存储空间来存储指针

#### 2.3.2 单链表的实现

**单链表的结构定义**：

```c
typedef struct LNode {
    ElemType data;  // 数据域
    struct LNode *next;  // 指针域
} LNode, *LinkList;
```

#### 2.3.3 单链表的插入

**前插法**：在指定节点之前插入新节点

```c
bool InsertPriorNode(LNode *p, ElemType e) {
    if (p == NULL)
        return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL)
        return false;
    s->data = p->data;
    s->next = p->next;
    p->data = e;
    p->next = s;
    return true;
}
```

**后插法**：在指定节点之后插入新节点

```c
bool InsertNextNode(LNode *p, ElemType e) {
    if (p == NULL)
        return false;
    LNode *s = (LNode *)malloc(sizeof(LNode));
    if (s == NULL)
        return false;
    s->data = e;
    s->next = p->next;
    p->next = s;
    return true;
}
```

#### 2.3.4 单链表的删除

**删除指定节点的后继节点**：

```c
bool DeleteNextNode(LNode *p) {
    if (p == NULL || p->next == NULL)
        return false;
    LNode *q = p->next;
    p->next = q->next;
    free(q);
    return true;
}
```

**删除指定节点**：

```c
bool DeleteNode(LNode *p) {
    if (p == NULL)
        return false;
    if (p->next == NULL) {
        free(p);
        p = NULL;
        return true;
    }
    LNode *q = p->next;
    p->data = q->data;
    p->next = q->next;
    free(q);
    return true;
}
```

#### 2.3.5 单链表的查找

**按位查找**：

```c
LNode *GetElem(LinkList L, int i) {
    if (i < 1)
        return NULL;
    int j = 1;
    LNode *p = L->next;
    while (p != NULL && j < i) {
        p = p->next;
        j++;
    }
    return p;
}
```

**按值查找**：

```c
LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;
    while (p != NULL && p->data != e)
        p = p->next;
    return p;
}
```

#### 2.3.6 单链表的建立

**头插法**：

```c
LinkList List_HeadInsert(LinkList &L) {
    L = (LinkList)malloc(sizeof(LNode));
    L->next = NULL;
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        s->next = L->next;
        L->next = s;
        scanf("%d", &x);
    }
    return L;
}
```

**尾插法**：

```c
LinkList List_TailInsert(LinkList &L) {
    L = (LinkList)malloc(sizeof(LNode));
    L->next = NULL;
    LNode *r = L;
    ElemType x;
    scanf("%d", &x);
    while (x != 9999) {
        LNode *s = (LNode *)malloc(sizeof(LNode));
        s->data = x;
        r->next = s;
        r = s;
        scanf("%d", &x);
    }
    r->next = NULL;
    return L;
}
```

#### 2.3.7 双链表

**双链表的结构定义**：

```c
typedef struct DNode {
    ElemType data;  // 数据域
    struct DNode *prior, *next;  // 前驱和后继指针
} DNode, *DLinkList;
```

**双链表的插入**：

```c
bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL)
        return false;
    s->next = p->next;
    if (p->next != NULL)
        p->next->prior = s;
    s->prior = p;
    p->next = s;
    return true;
}
```

**双链表的删除**：

```c
bool DeleteNextDNode(DNode *p) {
    if (p == NULL)
        return false;
    DNode *q = p->next;
    if (q == NULL)
        return false;
    p->next = q->next;
    if (q->next != NULL)
        q->next->prior = p;
    free(q);
    return true;
}
```

#### 2.3.8 循环链表

**循环单链表**：最后一个节点的指针指向头节点

**循环双链表**：最后一个节点的next指针指向头节点，头节点的prior指针指向最后一个节点

#### 2.3.9 静态链表

**静态链表**：用数组来模拟链表，每个数组元素包含数据域和指针域（数组下标）

**静态链表的结构定义**：

```c
#define MaxSize 100

typedef struct {
    ElemType data;  // 数据域
    int next;  // 指针域
} SLinkList[MaxSize];
```

#### 2.3.10 顺序表和链表的比较

| 比较项 | 顺序表 | 链表 |
| --- | --- | --- |
| 存储方式 | 顺序存储 | 链式存储 |
| 访问方式 | 随机存取 | 顺序存取 |
| 插入/删除 | 需要移动元素，时间复杂度O(n) | 不需要移动元素，时间复杂度O(1) |
| 空间利用 | 存储密度高 | 需要额外的存储空间来存储指针 |
| 适用场景 | 频繁访问，较少插入/删除 | 频繁插入/删除，较少访问 |

## 第三章 栈和队列

### 3.1 栈

#### 3.1.1 栈的基本概念

**栈**：是一种特殊的线性表，仅允许在表的一端进行插入和删除操作，这一端称为栈顶，另一端称为栈底。

**栈的特点**：后进先出（LIFO）

#### 3.1.2 栈的基本操作

1. **InitStack(&S)**：初始化栈。构造一个空栈S。
2. **StackEmpty(S)**：判空操作。若S为空栈，则返回true，否则返回false。
3. **Push(&S, x)**：进栈操作。将元素x压入栈S的栈顶。
4. **Pop(&S, &x)**：出栈操作。从栈S中弹出栈顶元素，并将其值赋给x。
5. **GetTop(S, &x)**：获取栈顶元素。将栈S的栈顶元素赋给x，但不弹出。
6. **DestroyStack(&S)**：销毁栈。释放栈S所占用的内存空间。

#### 3.1.3 栈的顺序存储实现

**顺序栈的结构定义**：

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 数组存储元素
    int top;  // 栈顶指针
} SqStack;
```

**顺序栈的基本操作**：

1. **初始化栈**：

```c
void InitStack(SqStack &S) {
    S.top = -1;  // 栈顶指针初始化为-1
}
```

2. **判空**：

```c
bool StackEmpty(SqStack S) {
    return S.top == -1;
}
```

3. **进栈**：

```c
bool Push(SqStack &S, ElemType x) {
    if (S.top == MaxSize - 1)  // 栈满
        return false;
    S.data[++S.top] = x;  // 栈顶指针加1，然后存入元素
    return true;
}
```

4. **出栈**：

```c
bool Pop(SqStack &S, ElemType &x) {
    if (S.top == -1)  // 栈空
        return false;
    x = S.data[S.top--];  // 取出栈顶元素，然后栈顶指针减1
    return true;
}
```

5. **获取栈顶元素**：

```c
bool GetTop(SqStack S, ElemType &x) {
    if (S.top == -1)  // 栈空
        return false;
    x = S.data[S.top];  // 取出栈顶元素
    return true;
}
```

#### 3.1.4 栈的链式存储

**链栈的结构定义**：

```c
typedef struct StackNode {
    ElemType data;  // 数据域
    struct StackNode *next;  // 指针域
} StackNode, *LinkStack;
```

**链栈的基本操作**：

1. **初始化栈**：

```c
void InitStack(LinkStack &S) {
    S = NULL;  // 栈顶指针初始化为NULL
}
```

2. **判空**：

```c
bool StackEmpty(LinkStack S) {
    return S == NULL;
}
```

3. **进栈**：

```c
bool Push(LinkStack &S, ElemType x) {
    StackNode *p = (StackNode *)malloc(sizeof(StackNode));
    if (p == NULL)  // 内存分配失败
        return false;
    p->data = x;
    p->next = S;
    S = p;  // 栈顶指针指向新节点
    return true;
}
```

4. **出栈**：

```c
bool Pop(LinkStack &S, ElemType &x) {
    if (S == NULL)  // 栈空
        return false;
    StackNode *p = S;
    x = p->data;
    S = S->next;
    free(p);
    return true;
}
```

5. **获取栈顶元素**：

```c
bool GetTop(LinkStack S, ElemType &x) {
    if (S == NULL)  // 栈空
        return false;
    x = S->data;
    return true;
}
```

### 3.2 队列

#### 3.2.1 队列的基本概念

**队列**：是一种特殊的线性表，仅允许在表的一端进行插入操作，在另一端进行删除操作。插入的一端称为队尾，删除的一端称为队头。

**队列的特点**：先进先出（FIFO）

#### 3.2.2 队列的基本操作

1. **InitQueue(&Q)**：初始化队列。构造一个空队列Q。
2. **QueueEmpty(Q)**：判空操作。若Q为空队列，则返回true，否则返回false。
3. **EnQueue(&Q, x)**：入队操作。将元素x加入队列Q的队尾。
4. **DeQueue(&Q, &x)**：出队操作。从队列Q中取出队头元素，并将其值赋给x。
5. **GetHead(Q, &x)**：获取队头元素。将队列Q的队头元素赋给x，但不删除。
6. **DestroyQueue(&Q)**：销毁队列。释放队列Q所占用的内存空间。

#### 3.2.3 队列的顺序存储实现

**顺序队列的结构定义**：

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 数组存储元素
    int front, rear;  // 队头和队尾指针
} SqQueue;
```

**顺序队列的基本操作**：

1. **初始化队列**：

```c
void InitQueue(SqQueue &Q) {
    Q.front = Q.rear = 0;  // 队头和队尾指针初始化为0
}
```

2. **判空**：

```c
bool QueueEmpty(SqQueue Q) {
    return Q.front == Q.rear;
}
```

3. **入队**：

```c
bool EnQueue(SqQueue &Q, ElemType x) {
    if ((Q.rear + 1) % MaxSize == Q.front)  // 队列满
        return false;
    Q.data[Q.rear] = x;
    Q.rear = (Q.rear + 1) % MaxSize;  // 队尾指针加1
    return true;
}
```

4. **出队**：

```c
bool DeQueue(SqQueue &Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队列空
        return false;
    x = Q.data[Q.front];
    Q.front = (Q.front + 1) % MaxSize;  // 队头指针加1
    return true;
}
```

5. **获取队头元素**：

```c
bool GetHead(SqQueue Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队列空
        return false;
    x = Q.data[Q.front];
    return true;
}
```

#### 3.2.4 队列的链式存储实现

**链队列的结构定义**：

```c
typedef struct LinkNode {
    ElemType data;  // 数据域
    struct LinkNode *next;  // 指针域
} LinkNode;

typedef struct {
    LinkNode *front, *rear;  // 队头和队尾指针
} LinkQueue;
```

**链队列的基本操作**：

1. **初始化队列**：

```c
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = (LinkNode *)malloc(sizeof(LinkNode));
    Q.front->next = NULL;
}
```

2. **判空**：

```c
bool QueueEmpty(LinkQueue Q) {
    return Q.front == Q.rear;
}
```

3. **入队**：

```c
void EnQueue(LinkQueue &Q, ElemType x) {
    LinkNode *s = (LinkNode *)malloc(sizeof(LinkNode));
    s->data = x;
    s->next = NULL;
    Q.rear->next = s;
    Q.rear = s;
}
```

4. **出队**：

```c
bool DeQueue(LinkQueue &Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队列空
        return false;
    LinkNode *p = Q.front->next;
    x = p->data;
    Q.front->next = p->next;
    if (Q.rear == p)  // 若队尾就是出队的节点
        Q.rear = Q.front;
    free(p);
    return true;
}
```

5. **获取队头元素**：

```c
bool GetHead(LinkQueue Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队列空
        return false;
    x = Q.front->next->data;
    return true;
}
```

#### 3.2.5 双端队列

**双端队列**：是一种允许在两端进行插入和删除操作的队列。

### 3.3 栈与队列的应用

#### 3.3.1 栈在括号匹配中的应用

**算法思路**：
1. 初始化一个空栈
2. 遍历表达式中的每个字符
3. 如果是左括号，入栈
4. 如果是右括号，出栈并检查是否匹配
5. 遍历结束后，检查栈是否为空

**代码实现**：

```c
bool bracketCheck(char *str) {
    SqStack S;
    InitStack(S);
    for (int i = 0; str[i] != '\0'; i++) {
        if (str[i] == '(' || str[i] == '[' || str[i] == '{') {
            Push(S, str[i]);
        } else {
            if (StackEmpty(S))  // 右括号多
                return false;
            char topElem;
            Pop(S, topElem);
            if (str[i] == ')' && topElem != '(')
                return false;
            if (str[i] == ']' && topElem != '[')
                return false;
            if (str[i] == '}' && topElem != '{')
                return false;
        }
    }
    return StackEmpty(S);  // 检查左括号是否多
}
```

#### 3.3.2 栈在表达式求值中的应用

**算法思路**：
1. 初始化两个栈：操作数栈和运算符栈
2. 遍历表达式中的每个字符
3. 如果是操作数，入操作数栈
4. 如果是运算符，比较其与运算符栈顶运算符的优先级
   - 如果优先级更高，入栈
   - 如果优先级更低或相等，弹出运算符栈顶运算符，并从操作数栈中弹出两个操作数进行计算，结果入操作数栈
5. 遍历结束后，弹出运算符栈中剩余的运算符，依次计算
6. 操作数栈中最后剩下的元素就是表达式的结果

#### 3.3.3 栈在递归中的应用

**递归的特点**：
- 函数调用时，系统会自动为每一层调用分配栈帧
- 递归结束时，系统会自动释放栈帧
- 递归的深度受栈大小的限制

**递归的优缺点**：
- 优点：代码简洁，可读性高
- 缺点：可能导致栈溢出，时间和空间复杂度较高

#### 3.3.4 队列的应用

**队列的应用场景**：
- 广度优先搜索
- 消息队列
- 任务调度
- 缓冲区管理

### 3.4 特殊矩阵的压缩存储

#### 3.4.1 数组的存储

**数组的存储方式**：
- 行优先存储：先存储第一行，再存储第二行，依此类推
- 列优先存储：先存储第一列，再存储第二列，依此类推

**数组元素的地址计算**：
- 行优先：LOC(i,j) = LOC(0,0) + (i * n + j) * L
- 列优先：LOC(i,j) = LOC(0,0) + (j * m + i) * L

其中，m是行数，n是列数，L是每个元素的存储长度。

#### 3.4.2 对称矩阵的压缩存储

**对称矩阵**：对于n阶矩阵A，若满足A[i][j] = A[j][i]（0≤i,j<n），则称为对称矩阵。

**压缩存储**：只存储主对角线及其以下的元素，共n(n+1)/2个元素。

**地址计算**：
- 当i≥j时，LOC(i,j) = LOC(0,0) + (i(i+1)/2 + j) * L
- 当i<j时，LOC(i,j) = LOC(j,i)

#### 3.4.3 三角矩阵的压缩存储

**上三角矩阵**：主对角线以下的元素均为常数c
**下三角矩阵**：主对角线以上的元素均为常数c

**压缩存储**：只存储主对角线及其以上（或以下）的元素，加上常数c，共n(n+1)/2 + 1个元素。

#### 3.4.4 三对角矩阵的压缩存储

**三对角矩阵**：除主对角线及其上下相邻的两条对角线外，其余元素均为0。

**压缩存储**：只存储三对角线上的元素，共3n-2个元素。

**地址计算**：LOC(i,j) = LOC(0,0) + (2i + j) * L

#### 3.4.5 稀疏矩阵的压缩存储

**稀疏矩阵**：非零元素个数远少于零元素个数的矩阵。

**压缩存储方式**：
1. 三元组表：存储非零元素的行号、列号和值
2. 十字链表：每个非零元素对应一个节点，节点包含行指针、列指针、值

## 第四章 串

### 4.1 串的基本概念

**串**：是由零个或多个字符组成的有限序列，又称为字符串。

**串的相关概念**：
- 空串：长度为0的串
- 子串：串中任意个连续的字符组成的子序列
- 主串：包含子串的串
- 字符在串中的位置：字符在串中的序号
- 子串在主串中的位置：子串的第一个字符在主串中的位置

### 4.2 串的基本操作

1. **StrAssign(&T, chars)**：赋值操作。把串T赋值为chars。
2. **StrCopy(&T, S)**：复制操作。把串S复制到串T。
3. **StrEmpty(S)**：判空操作。若S为空串，则返回true，否则返回false。
4. **StrLength(S)**：求串长。返回串S的长度。
5. **StrCompare(S, T)**：比较操作。若S>T，返回值>0；若S=T，返回0；若S<T，返回值<0。
6. **Concat(&T, S1, S2)**：串联接。用T返回由S1和S2联接而成的新串。
7. **SubString(&Sub, S, pos, len)**：求子串。用Sub返回串S的第pos个字符起长度为len的子串。
8. **Index(S, T)**：定位操作。若主串S中存在与串T值相同的子串，则返回它在主串S中第一次出现的位置；否则函数值为0。
9. **ClearString(&S)**：清空操作。将S清为空串。
10. **DestroyString(&S)**：销毁串。释放串S占用的存储空间。

### 4.3 串的存储实现

#### 4.3.1 静态数组实现

**定长顺序存储**：

```c
#define MaxSize 255
typedef struct {
    char ch[MaxSize];  // 存储串的数组
    int length;  // 串的长度
} SString;
```

**堆分配存储**：

```c
typedef struct {
    char *ch;  // 指向存储串的数组的指针
    int length;  // 串的长度
} HString;
```

#### 4.3.2 基本操作的实现

**串的比较**：

```c
int StrCompare(SString S, SString T) {
    for (int i = 1; i <= S.length && i <= T.length; i++) {
        if (S.ch[i] != T.ch[i])
            return S.ch[i] - T.ch[i];
    }
    return S.length - T.length;
}
```

**求子串**：

```c
bool SubString(SString &Sub, SString S, int pos, int len) {
    if (pos < 1 || pos > S.length || len < 0 || pos + len - 1 > S.length)
        return false;
    for (int i = 1; i <= len; i++)
        Sub.ch[i] = S.ch[pos + i - 1];
    Sub.length = len;
    return true;
}
```

**串的连接**：

```c
bool Concat(SString &T, SString S1, SString S2) {
    if (S1.length + S2.length > MaxSize)
        return false;
    for (int i = 1; i <= S1.length; i++)
        T.ch[i] = S1.ch[i];
    for (int i = 1; i <= S2.length; i++)
        T.ch[S1.length + i] = S2.ch[i];
    T.length = S1.length + S2.length;
    return true;
}
```

### 4.4 串的朴素模式匹配

**朴素模式匹配算法**：

```c
int Index(SString S, SString T) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            i = i - j + 2;  // 主串指针回溯
            j = 1;  // 模式串指针重置
        }
    }
    if (j > T.length)
        return i - T.length;
    else
        return 0;
}
```

**时间复杂度**：最好情况O(n+m)，最坏情况O(nm)，其中n是主串长度，m是模式串长度。

### 4.5 KPM算法

**KPM算法**：通过预处理模式串，避免主串指针的回溯，提高匹配效率。

**部分匹配表（next数组）**：

```c
void getNext(SString T, int next[]) {
    int i = 1, j = 0;
    next[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j];
        }
    }
}
```

**KPM算法**：

```c
int Index_KMP(SString S, SString T, int next[]) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (j == 0 || S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            j = next[j];
        }
    }
    if (j > T.length)
        return i - T.length;
    else
        return 0;
}
```

**时间复杂度**：O(n+m)，其中n是主串长度，m是模式串长度。

## 第五章 树

### 5.1 树的概念

#### 5.1.1 树的基本定义

**树**：是n（n≥0）个结点的有限集合。当n=0时，称为空树；当n>0时，有且仅有一个特定的称为根的结点，其余结点可分为m（m≥0）个互不相交的有限集合，每个集合本身又是一棵树，称为根的子树。

**树的相关概念**：
- 结点的度：结点拥有的子树数
- 树的度：树中结点的最大度数
- 叶子结点：度为0的结点
- 分支结点：度大于0的结点
- 孩子结点：结点的子树的根
- 双亲结点：孩子结点的父结点
- 兄弟结点：同一双亲的孩子结点
- 祖先结点：从根到该结点所经分支上的所有结点
- 子孙结点：以该结点为根的子树中的所有结点
- 层次：根为第1层，根的孩子为第2层，依此类推
- 树的高度：树中结点的最大层次
- 有序树：树中结点的子树从左到右是有次序的
- 无序树：树中结点的子树从左到右是无次序的
- 森林：m（m≥0）棵互不相交的树的集合

#### 5.1.2 树的常考性质

1. 树中的结点数等于所有结点的度数之和加1
2. 度为m的树中，第i层最多有m^(i-1)个结点
3. 高度为h的m叉树最多有(m^h - 1)/(m - 1)个结点
4. 具有n个结点的m叉树的最小高度为⌈log_m(n(m-1)+1)⌉

### 5.2 二叉树

#### 5.2.1 二叉树的定义

**二叉树**：是n（n≥0）个结点的有限集合，它或者是空集，或者由一个根结点和两棵互不相交的、分别称为左子树和右子树的二叉树组成。

**二叉树的特点**：
- 每个结点最多有两棵子树
- 左子树和右子树是有次序的
- 即使只有一棵子树，也要区分是左子树还是右子树

#### 5.2.2 特殊二叉树

1. **满二叉树**：高度为h，且含有2^h - 1个结点的二叉树
2. **完全二叉树**：高度为h，有n个结点的二叉树，当且仅当其每一个结点都与高度为h的满二叉树中编号为1~n的结点一一对应
3. **二叉排序树**：左子树上所有结点的关键字均小于根结点的关键字，右子树上所有结点的关键字均大于根结点的关键字
4. **平衡二叉树**：树上任一结点的左子树和右子树的深度之差不超过1

#### 5.2.3 二叉树的性质

1. 非空二叉树的叶子结点数等于度为2的结点数加1，即n0 = n2 + 1
2. 非空二叉树的第i层最多有2^(i-1)个结点
3. 高度为h的二叉树最多有2^h - 1个结点
4. 具有n个结点的完全二叉树的高度为⌊log2n⌋ + 1
5. 对于完全二叉树中的编号为i的结点：
   - 若i>1，则其父结点的编号为⌊i/2⌋
   - 若2i≤n，则其左孩子的编号为2i
   - 若2i+1≤n，则其右孩子的编号为2i+1

### 5.3 二叉树的遍历和线索二叉树

#### 5.3.1 二叉树的先中后序遍历

**先序遍历**：根→左→右

```c
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);  // 访问根结点
        PreOrder(T->lchild);  // 递归遍历左子树
        PreOrder(T->rchild);  // 递归遍历右子树
    }
}
```

**中序遍历**：左→根→右

```c
void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);  // 递归遍历左子树
        visit(T);  // 访问根结点
        InOrder(T->rchild);  // 递归遍历右子树
    }
}
```

**后序遍历**：左→右→根

```c
void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);  // 递归遍历左子树
        PostOrder(T->rchild);  // 递归遍历右子树
        visit(T);  // 访问根结点
    }
}
```

#### 5.3.2 二叉树的层序遍历

**层序遍历**：从上到下，从左到右依次访问每个结点

```c
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        if (p->lchild != NULL)
            EnQueue(Q, p->lchild);
        if (p->rchild != NULL)
            EnQueue(Q, p->rchild);
    }
}
```

#### 5.3.3 由遍历序列构造二叉树

**由先序和中序遍历序列构造二叉树**：
1. 先序序列的第一个元素是根结点
2. 在中序序列中找到根结点的位置，左边是左子树的中序序列，右边是右子树的中序序列
3. 根据左子树的中序序列长度，在先序序列中找到左子树的先序序列和右子树的先序序列
4. 递归构造左子树和右子树

**由中序和后序遍历序列构造二叉树**：
1. 后序序列的最后一个元素是根结点
2. 在中序序列中找到根结点的位置，左边是左子树的中序序列，右边是右子树的中序序列
3. 根据左子树的中序序列长度，在后序序列中找到左子树的后序序列和右子树的后序序列
4. 递归构造左子树和右子树

#### 5.3.4 线索二叉树的概念

**线索二叉树**：利用二叉树的空指针域，将空的左指针指向其前驱，空的右指针指向其后继

**线索二叉树的结构定义**：

```c
typedef struct ThreadNode {
    ElemType data;  // 数据域
    struct ThreadNode *lchild, *rchild;  // 左孩子和右孩子指针
    int ltag, rtag;  // 标志位，0表示指向孩子，1表示指向线索
} ThreadNode, *ThreadTree;
```

#### 5.3.5 二叉树的线索化

**中序线索化**：

```c
ThreadNode *pre = NULL;  // 全局变量，指向当前结点的前驱

void InThread(ThreadTree &p) {
    if (p != NULL) {
        InThread(p->lchild);  // 递归线索化左子树
        if (p->lchild == NULL) {  // 左子树为空，建立前驱线索
            p->lchild = pre;
            p->ltag = 1;
        }
        if (pre != NULL && pre->rchild == NULL) {  // 前驱的右子树为空，建立后继线索
            pre->rchild = p;
            pre->rtag = 1;
        }
        pre = p;  // 更新pre
        InThread(p->rchild);  // 递归线索化右子树
    }
}

void CreateInThread(ThreadTree T) {
    pre = NULL;
    if (T != NULL) {
        InThread(T);
        if (pre->rchild == NULL)  // 处理最后一个结点
            pre->rtag = 1;
    }
}
```

#### 5.3.6 在线索二叉树中找前驱/后继

**中序线索二叉树中找后继**：
- 如果rtag=1，则rchild指向后继
- 如果rtag=0，则后继是右子树中最左下的结点

**中序线索二叉树中找前驱**：
- 如果ltag=1，则lchild指向前驱
- 如果ltag=0，则前驱是左子树中最右下的结点

### 5.5 应用

#### 5.5.1 二叉排序树

**二叉排序树的定义**：左子树上所有结点的关键字均小于根结点的关键字，右子树上所有结点的关键字均大于根结点的关键字

**二叉排序树的基本操作**：

1. **查找**：

```c
BiTree SearchBST(BiTree T, KeyType key) {
    if (T == NULL || T->data == key)
        return T;
    else if (key < T->data)
        return SearchBST(T->lchild, key);
    else
        return SearchBST(T->rchild, key);
}
```

2. **插入**：

```c
bool InsertBST(BiTree &T, KeyType key) {
    if (T == NULL) {
        T = (BiTree)malloc(sizeof(BiTNode));
        T->data = key;
        T->lchild = T->rchild = NULL;
        return true;
    } else if (key == T->data) {
        return false;
    } else if (key < T->data) {
        return InsertBST(T->lchild, key);
    } else {
        return InsertBST(T->rchild, key);
    }
}
```

3. **删除**：

```c
bool DeleteBST(BiTree &T, KeyType key) {
    if (T == NULL)
        return false;
    else {
        if (key == T->data) {
            return Delete(T);
        } else if (key < T->data) {
            return DeleteBST(T->lchild, key);
        } else {
            return DeleteBST(T->rchild, key);
        }
    }
}

bool Delete(BiTree &p) {
    if (p->rchild == NULL) {
        BiTree q = p;
        p = p->lchild;
        free(q);
    } else if (p->lchild == NULL) {
        BiTree q = p;
        p = p->rchild;
        free(q);
    } else {
        BiTree q = p, s = p->lchild;
        while (s->rchild != NULL) {
            q = s;
            s = s->rchild;
        }
        p->data = s->data;
        if (q != p)
            q->rchild = s->lchild;
        else
            q->lchild = s->lchild;
        free(s);
    }
    return true;
}
```

#### 5.5.2 平衡二叉树

**平衡二叉树的定义**：树上任一结点的左子树和右子树的深度之差不超过1

**平衡因子**：结点的左子树深度减去右子树深度的值

**平衡二叉树的调整**：
1. LL型：右旋
2. RR型：左旋
3. LR型：先左旋后右旋
4. RL型：先右旋后左旋

#### 5.5.3 哈夫曼树

**哈夫曼树的定义**：带权路径长度最小的二叉树

**哈夫曼算法**：
1. 将n个权值作为n个叶子结点，构造n棵只有一个结点的二叉树
2. 选择两棵权值最小的二叉树作为左右子树，构造一棵新的二叉树，新二叉树的权值为左右子树权值之和
3. 删除这两棵二叉树，将新二叉树加入到森林中
4. 重复步骤2和3，直到森林中只有一棵二叉树

**哈夫曼编码**：
- 左分支表示0，右分支表示1
- 从根到叶子的路径上的0和1组成的序列即为该叶子结点的编码

## 第六章 图

### 6.1 图的基本概念

**图**：是由顶点集合V和边集合E组成的一种数据结构，记为G=(V,E)。

**图的相关概念**：
- 顶点：图中的基本元素
- 边：顶点之间的连接
- 无向图：边没有方向的图
- 有向图：边有方向的图
- 完全图：任意两个顶点之间都有边相连的图
- 稀疏图：边数远小于完全图的图
- 稠密图：边数接近完全图的图
- 顶点的度：与顶点相连的边数
- 路径：顶点序列v1,v2,…,vk，其中vi和vi+1之间有边
- 路径长度：路径中的边数
- 简单路径：路径中顶点不重复的路径
- 回路：起点和终点相同的路径
- 连通图：任意两个顶点之间都有路径的图
- 连通分量：无向图中的极大连通子图
- 强连通图：有向图中任意两个顶点之间都有路径的图
- 强连通分量：有向图中的极大强连通子图
- 生成树：包含图中所有顶点的极小连通子图
- 生成森林：由生成树组成的森林

### 6.2 图的存储

#### 6.2.1 邻接矩阵

**邻接矩阵**：用二维数组表示顶点之间的关系

**无向图的邻接矩阵**：
- 对称矩阵
- 第i行（或第i列）的元素之和等于顶点i的度

**有向图的邻接矩阵**：
- 第i行的元素之和等于顶点i的出度

**邻接矩阵的结构定义**：

#### 6.2.4 图的基本操作

1. **InitGraph(&G, n)**：初始化图。构造一个有n个顶点的空图G。
2. **InsertArc(&G, v, w)**：插入边。在图G中插入一条从顶点v到顶点w的边。
3. **DeleteArc(&G, v, w)**：删除边。在图G中删除一条从顶点v到顶点w的边。
4. **LocateVex(G, u)**：定位顶点。在图G中找到顶点u的位置。
5. **GetVex(G, v)**：获取顶点值。返回图G中顶点v的值。
6. **PutVex(&G, v, value)**：设置顶点值。将图G中顶点v的值设置为value。
7. **FirstAdjVex(G, v)**：获取第一个邻接点。返回图G中顶点v的第一个邻接点。
8. **NextAdjVex(G, v, w)**：获取下一个邻接点。返回图G中顶点v的邻接点w的下一个邻接点。

### 6.3 图的遍历

#### 6.3.1 广度优先遍历

**广度优先遍历（BFS）**：

```c
void BFS(Graph G, int v) {
    bool visited[MAX_VERTEX_NUM];
    for (int i = 0; i < G.vexnum; i++)
        visited[i] = false;
    Queue Q;
    InitQueue(Q);
    visit(v);
    visited[v] = true;
    EnQueue(Q, v);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, v);
        for (int w = FirstAdjVex(G, v); w >= 0; w = NextAdjVex(G, v, w)) {
            if (!visited[w]) {
                visit(w);
                visited[w] = true;
                EnQueue(Q, w);
            }
        }
    }
}
```

#### 6.3.2 深度优先遍历

**深度优先遍历（DFS）**：

```c
void DFS(Graph G, int v) {
    visit(v);
    visited[v] = true;
    for (int w = FirstAdjVex(G, v); w >= 0; w = NextAdjVex(G, v, w)) {
        if (!visited[w]) {
            DFS(G, w);
        }
    }
}
```

### 6.4 图的应用

#### 6.4.1 最小生成树

**最小生成树**：在连通图中，所有生成树中权值和最小的生成树

**Prim算法**：
1. 初始化一个空的生成树
2. 选择一个起始顶点加入生成树
3. 重复以下步骤，直到生成树包含所有顶点：
   - 选择一条连接生成树内顶点和生成树外顶点的最小权值边
   - 将该边加入生成树
   - 将该边的生成树外顶点加入生成树

**Kruskal算法**：
1. 将所有边按权值从小到大排序
2. 初始化一个空的生成树
3. 遍历排序后的边，对于每条边：
   - 如果该边的两个顶点不在同一个连通分量中，将该边加入生成树
4. 直到生成树包含所有顶点

#### 6.4.2 无权图的单源最短路径问题——BFS算法

**BFS算法**：
1. 初始化距离数组，将起始顶点的距离设为0，其他顶点的距离设为无穷大
2. 将起始顶点入队
3. 当队列不为空时：
   - 出队一个顶点v
   - 遍历v的所有邻接点w
   - 如果w的距离为无穷大，将w的距离设为v的距离加1，并将w入队

#### 6.4.3 单源最短路径问题——Dijkstra算法

**Dijkstra算法**：
1. 初始化距离数组，将起始顶点的距离设为0，其他顶点的距离设为无穷大
2. 初始化一个集合S，包含起始顶点
3. 重复以下步骤，直到S包含所有顶点：
   - 从不在S中的顶点中选择一个距离最小的顶点u
   - 将u加入S
   - 遍历u的所有邻接点w
   - 如果通过u到w的距离比w当前的距离小，更新w的距离

#### 6.4.4 各顶点间的最短路径问题——Floyd算法

**Floyd算法**：
1. 初始化距离矩阵，初始值为各顶点间的直接距离
2. 对于每个顶点k，作为中间顶点：
   - 对于每个顶点i，作为起点：
     - 对于每个顶点j，作为终点：
       - 如果i→k→j的距离比i→j的距离小，更新i→j的距离

#### 6.4.5 有向无环图描述表达式

**有向无环图（DAG）**：没有回路的有向图

**用DAG描述表达式**：
- 操作数作为叶子结点
- 运算符作为内部结点
- 边表示操作数与运算符之间的关系

#### 6.4.6 拓扑排序

**拓扑排序**：将有向无环图中的顶点按照依赖关系排序

**拓扑排序算法**：
1. 计算每个顶点的入度
2. 将入度为0的顶点入队
3. 当队列不为空时：
   - 出队一个顶点v，输出
   - 遍历v的所有邻接点w
   - 将w的入度减1
   - 如果w的入度为0，将w入队
4. 如果输出的顶点数小于图中的顶点数，说明图中存在回路

#### 6.4.7 关键路径

**关键路径**：在带权有向无环图中，从源点到汇点的最长路径

**关键路径算法**：
1. 拓扑排序
2. 计算每个顶点的最早开始时间
3. 计算每个顶点的最迟开始时间
4. 计算每个活动的最早开始时间和最迟开始时间
5. 最早开始时间等于最迟开始时间的活动就是关键活动

## 第七章 查找

### 7.1 查找概念

**查找**：在数据集合中寻找满足特定条件的数据元素

**查找的基本概念**：
- 查找表：用于查找的数据集合
- 关键字：数据元素中用于标识元素的属性
- 主关键字：唯一标识数据元素的关键字
- 次关键字：不唯一标识数据元素的关键字
- 平均查找长度（ASL）：查找过程中平均比较的关键字次数

### 7.2 顺序查找

**顺序查找**：从表的一端开始，依次比较每个元素的关键字，直到找到目标元素或遍历完整个表

**顺序查找算法**：

```c
int SeqSearch(int a[], int n, int key) {
    for (int i = 0; i < n; i++) {
        if (a[i] == key)
            return i;
    }
    return -1;
}
```

**平均查找长度**：
- 成功：(n+1)/2
- 失败：n

### 7.3 折半查找

**折半查找**：在有序表中，每次将查找区间缩小一半，直到找到目标元素或确定目标元素不存在

**折半查找算法**：

```c
int BinarySearch(int a[], int n, int key) {
    int low = 0, high = n - 1;
    while (low <= high) {
        int mid = (low + high) / 2;
        if (a[mid] == key)
            return mid;
        else if (a[mid] > key)
            high = mid - 1;
        else
            low = mid + 1;
    }
    return -1;
}
```

**平均查找长度**：O(log2n)

### 7.4 分块查找

**分块查找**：将表分成若干块，块内无序，块间有序，先确定目标元素所在的块，然后在块内顺序查找

**分块查找算法**：
1. 建立索引表，每个索引项包含块的最大关键字和块的起始位置
2. 二分查找索引表，确定目标元素所在的块
3. 在块内顺序查找目标元素

**平均查找长度**：O(log2m + n/m)，其中m是块数

### 7.5 红黑树

#### 7.5.1 为什么要发明红黑树？

**红黑树**：是一种自平衡的二叉搜索树，它在插入和删除操作时通过变色和旋转来保持树的平衡

**红黑树的优点**：
- 插入和删除操作的时间复杂度为O(logn)
- 相比AVL树，红黑树的旋转操作更少
- 适用于频繁插入和删除的场景

#### 7.5.2 红黑树的定义

**红黑树的性质**：
1. 每个结点要么是红色，要么是黑色
2. 根结点是黑色
3. 每个叶结点（NIL结点）是黑色
4. 如果一个结点是红色，则其两个子结点都是黑色
5. 从任意一个结点到其每个叶结点的路径上，黑色结点的数量相同

#### 7.5.3 红黑树的插入

**红黑树的插入操作**：
1. 按照二叉搜索树的插入规则插入新结点
2. 将新结点标记为红色
3. 检查是否违反红黑树的性质，如果违反，进行调整

**调整操作**：
- 情况1：父结点是黑色，无需调整
- 情况2：父结点是红色，叔叔结点也是红色，将父结点和叔叔结点标记为黑色，祖父结点标记为红色，递归检查祖父结点
- 情况3：父结点是红色，叔叔结点是黑色，且新结点是父结点的右孩子，先左旋父结点，转换为情况4
- 情况4：父结点是红色，叔叔结点是黑色，且新结点是父结点的左孩子，将父结点标记为黑色，祖父结点标记为红色，右旋祖父结点

### 7.6 B树和B+树

#### 7.6.1 B树

**B树**：是一种多路平衡查找树，适用于外存存储

**B树的定义**：
- 每个结点最多有m个子树
- 根结点至少有2个子树（除非根结点是叶结点）
- 非根非叶结点至少有⌈m/2⌉个子树
- 所有叶结点在同一层
- 每个结点包含的关键字个数为子树个数减1

#### 7.6.2 B树的基本操作

**B树的插入操作**：
1. 从根结点开始，查找插入位置
2. 如果找到，插入失败
3. 如果没找到，在叶结点中插入关键字
4. 如果叶结点的关键字个数超过m-1，分裂叶结点
5. 向上递归检查父结点是否需要分裂

**B树的删除操作**：
1. 从根结点开始，查找删除位置
2. 如果没找到，删除失败
3. 如果找到，分情况处理：
   - 如果是叶结点，直接删除
   - 如果是非叶结点，用前驱或后继替换，然后删除前驱或后继
4. 如果删除后结点的关键字个数少于⌈m/2⌉-1，进行调整
5. 调整操作包括：向兄弟结点借关键字、合并结点

#### 7.6.3 B+树

**B+树**：是B树的一种变体，适用于范围查询

**B+树的特点**：
- 所有关键字都在叶结点中
- 非叶结点只存储关键字和子树指针
- 叶结点之间通过链表连接
- 根结点到每个叶结点的路径长度相同

#### 7.6.4 B树和B+树的比较

**B树和B+树的区别**：
- B树的关键字分布在整个树中，B+树的关键字只在叶结点中
- B树的非叶结点存储关键字和数据，B+树的非叶结点只存储关键字
- B+树的叶结点之间有链表连接，支持范围查询
- B+树的查询效率更稳定，因为所有查询都要到叶结点

### 7.7 散列查找及其性能分析

#### 7.7.1 散列表的基本概念

**散列表**：通过散列函数将关键字映射到存储位置的数据结构

**散列函数**：将关键字转换为存储位置的函数

**冲突**：不同的关键字通过散列函数得到相同的存储位置

**处理冲突的方法**：
- 开放定址法：线性探测、二次探测、双重散列
- 链地址法：将冲突的关键字链接成链表

#### 7.7.2 散列查找及性能分析

**散列查找算法**：
1. 通过散列函数计算关键字的存储位置
2. 如果该位置为空，查找失败
3. 如果该位置的关键字等于目标关键字，查找成功
4. 如果该位置的关键字不等于目标关键字，根据处理冲突的方法继续查找

**散列查找的性能**：
- 平均查找长度取决于散列函数的好坏和处理冲突的方法
- 理想情况下，平均查找长度为O(1)

## 第八章 排序

### 8.1 排序的基本概念

**排序**：将一组数据按照一定的顺序排列

**排序的基本概念**：
- 稳定排序：相同关键字的元素在排序前后的相对位置不变
- 不稳定排序：相同关键字的元素在排序前后的相对位置可能改变
- 内部排序：排序过程中数据全部在内存中
- 外部排序：排序过程中需要使用外部存储

### 8.2 插入排序

#### 8.2.1 直接插入排序

**直接插入排序**：将一个记录插入到已排序的有序表中，得到一个新的有序表

**直接插入排序算法**：

```c
void InsertSort(int a[], int n) {
    for (int i = 1; i < n; i++) {
        int temp = a[i];
        int j = i - 1;
        while (j >= 0 && a[j] > temp) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = temp;
    }
}
```

**时间复杂度**：
- 最好情况：O(n)
- 最坏情况：O(n²)
- 平均情况：O(n²)

**空间复杂度**：O(1)

**稳定性**：稳定

#### 8.2.2 折半插入排序

**折半插入排序**：在直接插入排序的基础上，使用折半查找来寻找插入位置

**折半插入排序算法**：

```c
void BinaryInsertSort(int a[], int n) {
    for (int i = 1; i < n; i++) {
        int temp = a[i];
        int low = 0, high = i - 1;
        while (low <= high) {
            int mid = (low + high) / 2;
            if (a[mid] > temp)
                high = mid - 1;
            else
                low = mid + 1;
        }
        for (int j = i - 1; j >= low; j--)
            a[j + 1] = a[j];
        a[low] = temp;
    }
}
```

**时间复杂度**：
- 最好情况：O(n logn)
- 最坏情况：O(n²)
- 平均情况：O(n²)

**空间复杂度**：O(1)

**稳定性**：稳定

#### 8.2.3 希尔排序

**希尔排序**：将整个待排序序列分割成若干子序列分别进行直接插入排序，待整个序列中的记录基本有序时，再对全体记录进行一次直接插入排序

**希尔排序算法**：

```c
void ShellSort(int a[], int n) {
    for (int d = n / 2; d > 0; d /= 2) {
        for (int i = d; i < n; i++) {
            int temp = a[i];
            int j = i - d;
            while (j >= 0 && a[j] > temp) {
                a[j + d] = a[j];
                j -= d;
            }
            a[j + d] = temp;
        }
    }
}
```

**时间复杂度**：
- 最好情况：O(n logn)
- 最坏情况：O(n²)
- 平均情况：O(n^1.3)

**空间复杂度**：O(1)

**稳定性**：不稳定

### 8.3 交换排序

#### 8.3.1 冒泡排序

**冒泡排序**：通过相邻元素的比较和交换，将最大的元素逐步冒泡到数组的末尾

**冒泡排序算法**：

```c
void BubbleSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (a[j] > a[j + 1]) {
                int temp = a[j];
                a[j] = a[j + 1];
                a[j + 1] = temp;
                flag = true;
            }
        }
        if (!flag)
            break;
    }
}
```

**时间复杂度**：
- 最好情况：O(n)
- 最坏情况：O(n²)
- 平均情况：O(n²)

**空间复杂度**：O(1)

**稳定性**：稳定

#### 8.3.2 快速排序

**快速排序**：选择一个基准元素，将数组分为两部分，小于基准的在左边，大于基准的在右边，然后递归排序左右两部分

**快速排序算法**：

```c
int Partition(int a[], int low, int high) {
    int pivot = a[low];
    while (low < high) {
        while (low < high && a[high] >= pivot)
            high--;
        a[low] = a[high];
        while (low < high && a[low] <= pivot)
            low++;
        a[high] = a[low];
    }
    a[low] = pivot;
    return low;
}

void QuickSort(int a[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(a, low, high);
        QuickSort(a, low, pivotpos - 1);
        QuickSort(a, pivotpos + 1, high);
    }
}
```

**时间复杂度**：
- 最好情况：O(n logn)
- 最坏情况：O(n²)
- 平均情况：O(n logn)

**空间复杂度**：O(logn)

**稳定性**：不稳定

### 8.4 选择排序

#### 8.4.1 简单选择排序

**简单选择排序**：每次从待排序序列中选择最小的元素，放到已排序序列的末尾

**简单选择排序算法**：

```c
void SelectSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min = i;
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[min])
                min = j;
        }
        if (min != i) {
            int temp = a[i];
            a[i] = a[min];
            a[min] = temp;
        }
    }
}
```

**时间复杂度**：
- 最好情况：O(n²)
- 最坏情况：O(n²)
- 平均情况：O(n²)

**空间复杂度**：O(1)

**稳定性**：不稳定

#### 8.4.2 堆排序

**堆排序**：利用堆这种数据结构来进行排序

**堆的性质**：
- 大根堆：每个结点的值都大于或等于其左右孩子的值
- 小根堆：每个结点的值都小于或等于其左右孩子的值

**堆排序算法**：

```c
void HeapAdjust(int a[], int s, int m) {
    int temp = a[s];
    for (int j = 2 * s + 1; j <= m; j = 2 * j + 1) {
        if (j < m && a[j] < a[j + 1])
            j++;
        if (temp >= a[j])
            break;
        a[s] = a[j];
        s = j;
    }
    a[s] = temp;
}

void HeapSort(int a[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--)
        HeapAdjust(a, i, n - 1);
    for (int i = n - 1; i > 0; i--) {
        int temp = a[0];
        a[0] = a[i];
        a[i] = temp;
        HeapAdjust(a, 0, i - 1);
    }
}
```

**时间复杂度**：
- 最好情况：O(n logn)
- 最坏情况：O(n logn)
- 平均情况：O(n logn)

**空间复杂度**：O(1)

**稳定性**：不稳定

### 8.5 归并排序

**归并排序**：将两个或多个有序表合并成一个新的有序表

**归并排序算法**：

```c
void Merge(int a[], int low, int mid, int high) {
    int *b = (int *)malloc((high - low + 1) * sizeof(int));
    int i = low, j = mid + 1, k = 0;
    while (i <= mid && j <= high) {
        if (a[i] <= a[j])
            b[k++] = a[i++];
        else
            b[k++] = a[j++];
    }
    while (i <= mid)
        b[k++] = a[i++];
    while (j <= high)
        b[k++] = a[j++];
    for (i = low, k = 0; i <= high; i++, k++)
        a[i] = b[k];
    free(b);
}

void MergeSort(int a[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(a, low, mid);
        MergeSort(a, mid + 1, high);
        Merge(a, low, mid, high);
    }
}
```

**时间复杂度**：
- 最好情况：O(n logn)
- 最坏情况：O(n logn)
- 平均情况：O(n logn)

**空间复杂度**：O(n)

**稳定性**：稳定

### 8.6 基数排序

**基数排序**：按照低位先排序，然后收集；再按照高位排序，然后再收集；依此类推，直到最高位

**基数排序算法**：

```c
void RadixSort(int a[], int n) {
    int max = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] > max)
            max = a[i];
    }
    int digits = 0;
    while (max > 0) {
        max /= 10;
        digits++;
    }
    int *b = (int *)malloc(n * sizeof(int));
    int count[10] = {0};
    for (int i = 0; i < digits; i++) {
        int divisor = pow(10, i);
        memset(count, 0, sizeof(count));
        for (int j = 0; j < n; j++) {
            int digit = (a[j] / divisor) % 10;
            count[digit]++;
        }
        for (int j = 1; j < 10; j++) {
            count[j] += count[j - 1];
        }
        for (int j = n - 1; j >= 0; j--) {
            int digit = (a[j] / divisor) % 10;
            b[count[digit] - 1] = a[j];
            count[digit]--;
        }
        for (int j = 0; j < n; j++) {
            a[j] = b[j];
        }
    }
    free(b);
}
```

**时间复杂度**：O(d(n+r))，其中d是位数，r是基数

**空间复杂度**：O(n+r)

**稳定性**：稳定

### 8.8 外部排序

#### 8.8.1 外部排序的基本概念和方法

**外部排序**：排序过程中需要使用外部存储设备的排序

**外部排序的基本方法**：
1. 生成初始归并段
2. 对归并段进行多路平衡归并
3. 重复步骤2，直到得到一个有序文件

#### 8.8.2 败者树

**败者树**：用于多路归并的一种数据结构，记录每一轮比较的败者，使下一轮比较只需与胜者比较

**败者树的优点**：减少比较次数，提高多路归并的效率

#### 8.8.3 置换-选择排序（生成初始归并段）

**置换-选择排序的过程**：
1. 从输入文件中读取m个记录到内存工作区
2. 选择其中最小的记录作为输出归并段的第一个记录
3. 从输入文件中读取下一个记录到内存工作区
4. 选择内存工作区中所有小于等于当前归并段最大记录的记录中的最小记录作为下一个输出记录
5. 重复步骤3和4，直到内存工作区中没有小于等于当前归并段最大记录的记录
6. 开始一个新的归并段，重复步骤2到5

#### 8.8.4 最佳归并树

**最佳归并树**：根据归并段的长度，构造一棵最小带权路径长度的归并树，以减少归并过程中的I/O次数

2. 按照哈夫曼算法构造归并树

    LinkNode *p = Q.front->next;
    x = p->data;
    Q.front->next = p->next;
    if (Q.rear == p)  // 若队尾就是出队的节点
        Q.rear = Q.front;
    free(p);
    return true;
}
```

5. **获取队头元素**：

```c
bool GetHead(LinkQueue Q, ElemType &x) {
    if (Q.front == Q.rear)  // 队列空
        return false;
    x = Q.front->next->data;
    return true;
}
```

### 3.3 栈与队列的应用

#### 3.3.1 栈在括号匹配中的应用

**算法思路**：
1. 初始化一个空栈
2. 遍历表达式中的每个字符
3. 如果是左括号，入栈
4. 如果是右括号，出栈并检查是否匹配
5. 遍历结束后，检查栈是否为空

**代码实现**：

```c
bool bracketCheck(char *str) {
    SqStack S;
    InitStack(S);
    for (int i = 0; str[i] != '\0'; i++) {
        if (str[i] == '(' || str[i] == '[' || str[i] == '{') {
            Push(S, str[i]);
        } else {
            if (StackEmpty(S))  // 右括号多
                return false;
            char topElem;
            Pop(S, topElem);
            if (str[i] == ')' && topElem != '(')
                return false;
            if (str[i] == ']' && topElem != '[')
                return false;
            if (str[i] == '}' && topElem != '{')
                return false;
        }
    }
    return StackEmpty(S);  // 检查左括号是否多
}
```

#### 3.3.2 栈在表达式求值中的应用

**算法思路**：
1. 初始化两个栈：操作数栈和运算符栈
2. 遍历表达式中的每个字符
3. 如果是操作数，入操作数栈
4. 如果是运算符，比较其与运算符栈顶运算符的优先级
   - 如果优先级更高，入栈
   - 如果优先级更低或相等，弹出运算符栈顶运算符，并从操作数栈中弹出两个操作数进行计算，结果入操作数栈
5. 遍历结束后，弹出运算符栈中剩余的运算符，依次计算
6. 操作数栈中最后剩下的元素就是表达式的结果

#### 3.3.3 栈在递归中的应用

**递归的特点**：
- 函数调用时，系统会自动为每一层调用分配栈帧
- 递归结束时，系统会自动释放栈帧
- 递归的深度受栈大小的限制

**递归的优缺点**：
- 优点：代码简洁，可读性高
- 缺点：可能导致栈溢出，时间和空间复杂度较高

### 3.4 特殊矩阵的压缩存储

#### 3.4.1 数组的存储

**数组的存储方式**：
- 行优先存储：先存储第一行，再存储第二行，依此类推
- 列优先存储：先存储第一列，再存储第二列，依此类推

**数组元素的地址计算**：
- 行优先：LOC(i,j) = LOC(0,0) + (i * n + j) * L
- 列优先：LOC(i,j) = LOC(0,0) + (j * m + i) * L

其中，m是行数，n是列数，L是每个元素的存储长度。

#### 3.4.2 对称矩阵的压缩存储

**对称矩阵**：对于n阶矩阵A，若满足A[i][j] = A[j][i]（0≤i,j<n），则称为对称矩阵。

**压缩存储**：只存储主对角线及其以下的元素，共n(n+1)/2个元素。

**地址计算**：
- 当i≥j时，LOC(i,j) = LOC(0,0) + (i(i+1)/2 + j) * L
- 当i<j时，LOC(i,j) = LOC(j,i)

#### 3.4.3 三角矩阵的压缩存储

**上三角矩阵**：主对角线以下的元素均为常数c
**下三角矩阵**：主对角线以上的元素均为常数c

**压缩存储**：只存储主对角线及其以上（或以下）的元素，加上常数c，共n(n+1)/2 + 1个元素。

#### 3.4.4 三对角矩阵的压缩存储

**三对角矩阵**：除主对角线及其上下相邻的两条对角线外，其余元素均为0。

**压缩存储**：只存储三对角线上的元素，共3n-2个元素。

**地址计算**：LOC(i,j) = LOC(0,0) + (2i + j) * L

#### 3.4.5 稀疏矩阵的压缩存储

**稀疏矩阵**：非零元素个数远少于零元素个数的矩阵。

**压缩存储方式**：
1. 三元组表：存储非零元素的行号、列号和值
2. 十字链表：每个非零元素对应一个节点，节点包含行指针、列指针、值

## 第四章 串

### 4.1 串的基本概念

**串**：是由零个或多个字符组成的有限序列，又称为字符串。

**串的相关概念**：
- 空串：长度为0的串
- 子串：串中任意个连续的字符组成的子序列
- 主串：包含子串的串
- 字符在串中的位置：字符在串中的序号
- 子串在主串中的位置：子串的第一个字符在主串中的位置

### 4.2 串的基本操作

1. **StrAssign(&T, chars)**：赋值操作。把串T赋值为chars。
2. **StrCopy(&T, S)**：复制操作。把串S复制到串T。
3. **StrEmpty(S)**：判空操作。若S为空串，则返回true，否则返回false。
4. **StrLength(S)**：求串长。返回串S的长度。
5. **StrCompare(S, T)**：比较操作。若S>T，返回值>0；若S=T，返回0；若S<T，返回值<0。
6. **Concat(&T, S1, S2)**：串联接。用T返回由S1和S2联接而成的新串。
7. **SubString(&Sub, S, pos, len)**：求子串。用Sub返回串S的第pos个字符起长度为len的子串。
8. **Index(S, T)**：定位操作。若主串S中存在与串T值相同的子串，则返回它在主串S中第一次出现的位置；否则函数值为0。
9. **ClearString(&S)**：清空操作。将S清为空串。
10. **DestroyString(&S)**：销毁串。释放串S占用的存储空间。

#### 4.3.1 静态数组实现

#### 4.3.2 基本操作的实现

**串的比较**：

```c
int StrCompare(SString S, SString T) {
    for (int i = 1; i <= S.length && i <= T.length; i++) {
        if (S.ch[i] != T.ch[i])
            return S.ch[i] - T.ch[i];
    }
    return S.length - T.length;
}
```

**求子串**：

```c
bool SubString(SString &Sub, SString S, int pos, int len) {
    if (pos < 1 || pos > S.length || len < 0 || pos + len - 1 > S.length)
        return false;
    for (int i = 1; i <= len; i++)
        Sub.ch[i] = S.ch[pos + i - 1];
    Sub.length = len;
    return true;
}
```

**串的连接**：

```c
bool Concat(SString &T, SString S1, SString S2) {
    if (S1.length + S2.length > MaxSize)
        return false;
    for (int i = 1; i <= S1.length; i++)
        T.ch[i] = S1.ch[i];
    for (int i = 1; i <= S2.length; i++)
        T.ch[S1.length + i] = S2.ch[i];
    T.length = S1.length + S2.length;
    return true;
}
```

### 4.4 串的朴素模式匹配

**朴素模式匹配算法**：

```c
int Index(SString S, SString T) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            i = i - j + 2;  // 主串指针回溯
            j = 1;  // 模式串指针重置
        }
    }
    if (j > T.length)
        return i - T.length;
    else
        return 0;
}
```

**时间复杂度**：最好情况O(n+m)，最坏情况O(nm)，其中n是主串长度，m是模式串长度。

### 4.5 KPM算法

**KPM算法**：通过预处理模式串，避免主串指针的回溯，提高匹配效率。

**部分匹配表（next数组）**：

```c
void getNext(SString T, int next[]) {
    int i = 1, j = 0;
    next[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            i++;
            j++;
            next[i] = j;
        } else {
            j = next[j];
        }
    }
}
```

**KPM算法**：

```c
int Index_KMP(SString S, SString T, int next[]) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (j == 0 || S.ch[i] == T.ch[j]) {
            i++;
            j++;
        } else {
            j = next[j];
        }
    }
    if (j > T.length)
        return i - T.length;
    else
        return 0;
}
```

**时间复杂度**：O(n+m)，其中n是主串长度，m是模式串长度。

## 第五章 树

### 5.1 树的概念

#### 5.1.1 树的基本定义

**树**：是n（n≥0）个结点的有限集合。当n=0时，称为空树；当n>0时，有且仅有一个特定的称为根的结点，其余结点可分为m（m≥0）个互不相交的有限集合，每个集合本身又是一棵树，称为根的子树。

**树的相关概念**：
- 结点的度：结点拥有的子树数
- 树的度：树中结点的最大度数
- 叶子结点：度为0的结点
- 分支结点：度大于0的结点
- 孩子结点：结点的子树的根
- 双亲结点：孩子结点的父结点
- 兄弟结点：同一双亲的孩子结点
- 祖先结点：从根到该结点所经分支上的所有结点
- 子孙结点：以该结点为根的子树中的所有结点
- 层次：根为第1层，根的孩子为第2层，依此类推
- 树的高度：树中结点的最大层次
- 有序树：树中结点的子树从左到右是有次序的
- 无序树：树中结点的子树从左到右是无次序的
- 森林：m（m≥0）棵互不相交的树的集合

#### 5.1.2 树的常考性质

1. 树中的结点数等于所有结点的度数之和加1
2. 度为m的树中，第i层最多有m^(i-1)个结点
3. 高度为h的m叉树最多有(m^h - 1)/(m - 1)个结点
4. 具有n个结点的m叉树的最小高度为⌈log_m(n(m-1)+1)⌉

### 5.2 二叉树

#### 5.2.1 二叉树的定义

**二叉树**：是n（n≥0）个结点的有限集合，它或者是空集，或者由一个根结点和两棵互不相交的、分别称为左子树和右子树的二叉树组成。

**二叉树的特点**：
- 每个结点最多有两棵子树
- 左子树和右子树是有次序的
- 即使只有一棵子树，也要区分是左子树还是右子树

#### 5.2.2 特殊二叉树

1. **满二叉树**：高度为h，且含有2^h - 1个结点的二叉树
2. **完全二叉树**：高度为h，有n个结点的二叉树，当且仅当其每一个结点都与高度为h的满二叉树中编号为1~n的结点一一对应
3. **二叉排序树**：左子树上所有结点的关键字均小于根结点的关键字，右子树上所有结点的关键字均大于根结点的关键字
4. **平衡二叉树**：树上任一结点的左子树和右子树的深度之差不超过1

#### 5.2.3 二叉树的性质

1. 非空二叉树的叶子结点数等于度为2的结点数加1，即n0 = n2 + 1
2. 非空二叉树的第i层最多有2^(i-1)个结点
3. 高度为h的二叉树最多有2^h - 1个结点
4. 具有n个结点的完全二叉树的高度为⌊log2n⌋ + 1
5. 对于完全二叉树中的编号为i的结点：
   - 若i>1，则其父结点的编号为⌊i/2⌋
   - 若2i≤n，则其左孩子的编号为2i
   - 若2i+1≤n，则其右孩子的编号为2i+1

#### 5.2.4 二叉树存储实现

**顺序存储**：

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 存储二叉树的数组
    int n;  // 二叉树的结点数
} SqBiTree;
```

**链式存储**：

```c

```

### 5.3 二叉树的遍历和线索二叉树

#### 5.3.1 二叉树的先中后序遍历

**先序遍历**：根→左→右

```c
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);  // 访问根结点
        PreOrder(T->lchild);  // 递归遍历左子树
        PreOrder(T->rchild);  // 递归遍历右子树
    }
}
```

**中序遍历**：左→根→右

```c
void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);  // 递归遍历左子树
        visit(T);  // 访问根结点
        InOrder(T->rchild);  // 递归遍历右子树
    }
}
```

**后序遍历**：左→右→根

```c
void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);  // 递归遍历左子树
        PostOrder(T->rchild);  // 递归遍历右子树
        visit(T);  // 访问根结点
    }
}
```

#### 5.3.2 二叉树的层序遍历

**层序遍历**：从上到下，从左到右依次访问每个结点

```c
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        if (p->lchild != NULL)
            EnQueue(Q, p->lchild);
        if (p->rchild != NULL)
            EnQueue(Q, p->rchild);
    }
}
```

#### 5.3.3 由遍历序列构造二叉树

**由先序和中序遍历序列构造二叉树**：
1. 先序序列的第一个元素是根结点
2. 在中序序列中找到根结点的位置，左边是左子树的中序序列，右边是右子树的中序序列
3. 根据左子树的中序序列长度，在先序序列中找到左子树的先序序列和右子树的先序序列
4. 递归构造左子树和右子树

**由中序和后序遍历序列构造二叉树**：
1. 后序序列的最后一个元素是根结点
2. 在中序序列中找到根结点的位置，左边是左子树的中序序列，右边是右子树的中序序列
3. 根据左子树的中序序列长度，在后序序列中找到左子树的后序序列和右子树的后序序列
4. 递归构造左子树和右子树

#### 5.3.4 线索二叉树的概念

**线索二叉树**：利用二叉树的空指针域，将空的左指针指向其前驱，空的右指针指向其后继

**线索二叉树的结构定义**：

```c
typedef struct ThreadNode {
    ElemType data;  // 数据域
    struct ThreadNode *lchild, *rchild;  // 左孩子和右孩子指针
    int ltag, rtag;  // 标志位，0表示指向孩子，1表示指向线索
} ThreadNode, *ThreadTree;
```

#### 5.3.5 二叉树的线索化

**中序线索化**：

```c
ThreadNode *pre = NULL;  // 全局变量，指向当前结点的前驱

void InThread(ThreadTree &p) {
    if (p != NULL) {
        InThread(p->lchild);  // 递归线索化左子树
        if (p->lchild == NULL) {  // 左子树为空，建立前驱线索
            p->lchild = pre;
            p->ltag = 1;
        }
        if (pre != NULL && pre->rchild == NULL) {  // 前驱的右子树为空，建立后继线索
            pre->rchild = p;
            pre->rtag = 1;
        }
        pre = p;  // 更新pre
        InThread(p->rchild);  // 递归线索化右子树
    }
}

void CreateInThread(ThreadTree T) {
    pre = NULL;
    if (T != NULL) {
        InThread(T);
        if (pre->rchild == NULL)  // 处理最后一个结点
            pre->rtag = 1;
    }
}
```

#### 5.3.6 在线索二叉树中找前驱/后继

**中序线索二叉树中找后继**：
- 如果rtag=1，则rchild指向后继
- 如果rtag=0，则后继是右子树中最左下的结点

**中序线索二叉树中找前驱**：
- 如果ltag=1，则lchild指向前驱
- 如果ltag=0，则前驱是左子树中最右下的结点

### 5.5 应用

#### 5.5.1 二叉排序树

**二叉排序树的定义**：左子树上所有结点的关键字均小于根结点的关键字，右子树上所有结点的关键字均大于根结点的关键字

**二叉排序树的基本操作**：

1. **查找**：

```c
BiTree SearchBST(BiTree T, KeyType key) {
    if (T == NULL || T->data == key)
        return T;
    else if (key < T->data)
        return SearchBST(T->lchild, key);
    else
        return SearchBST(T->rchild, key);
}
```

2. **插入**：

```c
bool InsertBST(BiTree &T, KeyType key) {
    if (T == NULL) {
        T = (BiTree)malloc(sizeof(BiTNode));
        T->data = key;
        T->lchild = T->rchild = NULL;
        return true;
    } else if (key == T->data) {
        return false;
    } else if (key < T->data) {
        return InsertBST(T->lchild, key);
    } else {
        return InsertBST(T->rchild, key);
    }
}

3. **删除**：

```c
bool DeleteBST(BiTree &T, KeyType key) {
    if (T == NULL)
        return false;
    else {
        if (key == T->data) {
            return Delete(T);
        } else if (key < T->data) {
            return DeleteBST(T->lchild, key);
        } else {
            return DeleteBST(T->rchild, key);
        }
    }
}

bool Delete(BiTree &p) {
    if (p->rchild == NULL) {
        BiTree q = p;
        p = p->lchild;
        free(q);
    } else if (p->lchild == NULL) {
        BiTree q = p;
        p = p->rchild;
        free(q);
    } else {
        BiTree q = p, s = p->lchild;
        while (s->rchild != NULL) {
            q = s;
            s = s->rchild;
        }
        p->data = s->data;
        if (q != p)
            q->rchild = s->lchild;
        else
            q->lchild = s->lchild;
        free(s);
    }
    return true;
}
```

#### 5.5.2 平衡二叉树

**平衡二叉树的定义**：树上任一结点的左子树和右子树的深度之差不超过1

**平衡因子**：结点的左子树深度减去右子树深度的值

**平衡二叉树的调整**：
1. LL型：右旋
2. RR型：左旋
3. LR型：先左旋后右旋
4. RL型：先右旋后左旋

#### 5.5.3 哈夫曼树

**哈夫曼树的定义**：带权路径长度最小的二叉树

**哈夫曼算法**：
1. 将n个权值作为n个叶子结点，构造n棵只有一个结点的二叉树
2. 选择两棵权值最小的二叉树作为左右子树，构造一棵新的二叉树，新二叉树的权值为左右子树权值之和
3. 删除这两棵二叉树，将新二叉树加入到森林中
4. 重复步骤2和3，直到森林中只有一棵二叉树

**哈夫曼编码**：
- 左分支表示0，右分支表示1
- 从根到叶子的路径上的0和1组成的序列即为该叶子结点的编码

## 第六章 图

### 6.1 图的基本概念

**图**：是由顶点集合V和边集合E组成的一种数据结构，记为G=(V,E)。

**图的相关概念**：
- 顶点：图中的基本元素
- 边：顶点之间的连接
- 无向图：边没有方向的图
- 有向图：边有方向的图
- 完全图：任意两个顶点之间都有边相连的图
- 稀疏图：边数远小于完全图的图
- 稠密图：边数接近完全图的图
- 顶点的度：与顶点相连的边数
- 路径：顶点序列v1,v2,…,vk，其中vi和vi+1之间有边
- 路径长度：路径中的边数
- 简单路径：路径中顶点不重复的路径
- 回路：起点和终点相同的路径
- 连通图：任意两个顶点之间都有路径的图
- 连通分量：无向图中的极大连通子图
- 强连通图：有向图中任意两个顶点之间都有路径的图
- 强连通分量：有向图中的极大强连通子图
- 生成树：包含图中所有顶点的极小连通子图
- 生成森林：由生成树组成的森林
